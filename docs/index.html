Why mcp-pcloud

Features

Everything you need to recover, restore, and manage your pCloud files through natural conversation.

🔐
Secure Token Auth
Reads your pCloud bearer token from ~/.config/pcloud/tokens.json or the MCP_PCLOUD_TOKEN env var — no hardcoded credentials.
🗑️
Trash Management
List every file in your pCloud trash and restore any of them by file ID, with a safety confirmation gate.
⏪
File Rewind
Browse the full version history of any file and recover older versions to a destination path of your choice.
🌍
EU & US Routing
Respects pCloud's dynamic API hostname — use api.pcloud.com for US accounts or eapi.pcloud.com for EU accounts.
⚡
TypeScript & MCP SDK
Built on TypeScript 5 and MCP SDK 1.27 with native fetch — no heavy dependencies, fast cold start.
🤖
Universal AI Support
Works out of the box with Claude Code CLI, Claude Desktop, Cursor, Windsurf, and VSCode via the MCP protocol.
Get going fast

Quick Start

Add the server to Claude Code in one command, then start asking about your files. 

# Add to Claude Code CLI
claude mcp add mcp-pcloud -- npx -y @kud/mcp-pcloud

# Or add to Claude Desktop (claude_desktop_config.json)
{
  "mcpServers": {
    "mcp-pcloud": {
      "command": "npx",
      "args": ["-y", "@kud/mcp-pcloud"]
    }
  }
}

# Restart your AI client — pCloud tools appear automatically
✔ mcp-pcloud registered
Available tools

Tool Reference

Four tools covering pCloud trash management and file rewind recovery.

list_trash List all files currently in the pCloud trash, including file IDs, names, sizes, and deletion dates.
restore_from_trash Restore a file from trash by its file ID. Requires confirm: true as a safety gate before the operation executes.
list_rewind_events List all version history events for a given file path, returning a timeline of previous versions with their IDs and timestamps.
restore_from_rewind Restore a file from rewind history to a new destination path. Requires a rewind file ID and confirm: true before executing.
Configuration

Authentication

The server resolves your pCloud token automatically — no manual setup needed if you use the config file.

Key Type Default Description
MCP_PCLOUD_TOKEN env var — pCloud bearer access token. Takes priority over the config file.
~/.config/pcloud/tokens.json file — JSON file with access_token and optional hostname fields.
hostname string api.pcloud.com API hostname. Use eapi.pcloud.com for EU accounts.
